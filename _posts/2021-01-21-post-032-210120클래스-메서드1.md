자바(Java)와 같은 객체지향 프로그래밍(OOP) 언어의 핵심은 실세계의 사물이나 개념을 컴퓨터가 이해할 수 있도록 구조화하는 것입니다. 자바에서는 이를 위해 **클래스(Class)**라는 문법을 제공하여 개발자가 직접 새로운 데이터 타입을 정의할 수 있게 합니다.

이번 포스팅에서는 클래스를 이용한 커스텀 데이터 타입 정의 방법과, 이 과정에서 반드시 구별하여 이해해야 하는 **인스턴스(Instance)**와 **레퍼런스(Reference)**의 물리적 개념 및 메모리 상의 구조에 대해 정리해 보겠습니다.

---

## 1. 클래스를 활용한 새로운 데이터 타입 정의

기본적으로 자바는 `int`, `double`, `boolean` 등 8가지의 기본 자료형(Primitive Type)을 지원합니다. 하지만 실무에서 다루는 비즈니스 데이터(예: 회원 정보, 상품 정보 등)는 이러한 단일 타입으로 설명하기 어렵습니다.

### 예시: 회원 데이터 타입 정의
```java
// Member라는 새로운 참조 타입을 정의합니다.
public class Member {
    String id;
    String name;
    int point;
}
```
* **이점**: 여러 가지 종류의 이종(異種) 데이터 타입을 하나의 묶음(`Member`)으로 추상화할 수 있습니다. 
* 기본형 배열(예: `int[]`)은 단 하나의 데이터 형식만 일렬로 담을 수 있지만, 클래스 배열(`Member[]`)을 선언하면 다양한 정보(id, name, point)를 내포한 복합 구조 객체들을 하나의 배열로 묶어 관리할 수 있어 데이터 구조화 능력이 획기적으로 향상됩니다.

---

## 2. 인스턴스(Instance) vs 레퍼런스(Reference)

많은 입문자가 '객체', '인스턴스', '레퍼런스(참조)'라는 단어를 혼용하여 사용합니다. 하지만 메모리 관점에서는 이 둘을 명확하게 선을 그어 이해해야 합니다.

### 1) 인스턴스 (Instance)
* **정의**: 클래스라는 설계도를 바탕으로 **힙(Heap) 메모리 영역에 실제로 생성된 구체적인 객체 물리 자원** 자체를 의미합니다.
* **특징**: `new` 연산자가 실행될 때마다 메모리 상에 개별적인 공간을 차용하여 데이터의 실질적인 값(속성 필드값들)을 저장합니다.

### 2) 레퍼런스 (Reference)
* **정의**: 생성된 인스턴스의 **메모리 시작 주소(참조값)를 가리키는 변수**를 의미합니다.
* **특징**: 레퍼런스는 실제 데이터 값 자체를 담고 있지 않으며, 스택(Stack) 영역에 존재하면서 힙 영역에 생성된 인스턴스를 포인팅(Pointing)합니다.

---

## 3. 클래스 인스턴스 생성 및 레퍼런스 배열 다루기

### 1) 단일 인스턴스 생성
```java
// myMember는 레퍼런스 변수 (스택 영역)
// new Member()에 의해 힙 영역에 실제 Member 인스턴스가 할당되고 그 주소가 myMember에 대입됨
Member myMember = new Member();
```

### 2) 레퍼런스 배열 생성
여러 명의 회원 객체를 그룹으로 다루기 위해 레퍼런스 배열을 생성합니다.
```java
// 5개의 Member 객체 주소를 담을 수 있는 레퍼런스 배열을 선언
Member[] members = new Member[5];
```
* **주의**: 이 시점에서 `members` 배열의 각 인덱스는 `null` 주소값으로만 가득 차 있습니다. 실제 인스턴스들이 배열 내부에 배치된 상태가 아닙니다.

### 3) 인스턴스와 레퍼런스 배열 결합
각 레퍼런스 배열 방에 실제 개별 인스턴스 주소를 할당해 줍니다.
```java
members[0] = new Member();
members[0].id = "user01";
members[0].name = "김철수";

members[1] = new Member();
members[1].id = "user02";
members[1].name = "이영희";
```

이중 참조 구조를 명확히 그릴 수 있어야 `NullPointerException`과 같은 대표적인 런타임 예외를 예측하고 차단할 수 있습니다. 클래스라는 커스텀 데이터 타입 설계를 통해 비즈니스 복잡도를 현격히 낮추는 객체지향의 첫 단추를 견고히 꿰어 보시길 바랍니다.
