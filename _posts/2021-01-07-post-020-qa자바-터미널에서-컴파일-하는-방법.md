최근에는 IntelliJ나 Eclipse 같은 통합 개발 환경(IDE)이 빌드 및 실행을 모두 대신해 주기 때문에 개발자가 로컬 터미널에서 직접 자바 컴파일러 명령어를 다룰 일이 드물어졌습니다. 하지만 빌드 자동화 도구(Gradle, Maven)나 리눅스 배포 환경 등 로우 레벨(Low-level)에서의 동작을 온전히 디버깅하고 이해하려면 **CLI 환경에서의 자바 컴파일 및 클래스패스 바인딩**을 정확하게 파악하고 있어야 합니다.

이번 포스팅에서는 터미널 환경에서 `javac`와 `java` 명령어를 옵션별로 조작하는 구체적인 실무 방법에 대해 다뤄보겠습니다.

---

## 1. 패키지 경로를 포함하는 자바 컴파일: `javac -d`

자바 소스 파일(`.java`)을 실행 가능한 바이트코드 파일(`.class`)로 변환하기 위해 자바 컴파일러인 `javac` 명령어를 호출합니다.

### 1) 대상 소스 및 패키지 구조 예시
```
src/main/java/com/eomcs/lang/ex03/Exam0100.java (소스코드 위치)
```
소스 코드 상단에는 다음과 같은 패키지명이 선언되어 있을 것입니다.
```java
package com.eomcs.lang.ex03;
```

### 2) 컴파일 대상 및 경로 지정
자바 컴파일러는 패키지 선언문을 해석하여 물리적인 디렉토리 구조로 만들어 줍니다. 이때 컴파일된 결과물(.class)을 다른 폴더로 격리하여 저장하고 싶을 때 **`-d`** 옵션을 사용합니다.

```bash
# -d 옵션 뒤에 출력 폴더 경로를 명시합니다.
javac -d bin/main src/main/java/com/eomcs/lang/ex03/Exam0100.java
```
* **`-d bin/main`**: 컴파일에 성공하면 `bin/main` 폴더 하위에 패키지 구조인 `com/eomcs/lang/ex03/Exam0100.class` 파일 및 폴더 트리를 자동으로 구성하라는 지시입니다.
* **이점**: 소스 코드 폴더(`src`)와 빌드 결과물 폴더(`bin`)를 깔끔하게 분리할 수 있습니다.

---

## 2. 생성된 클래스 파일 실행과 클래스패스: `java -cp`

컴파일을 완료하여 생성된 바이트코드를 자바 가상 머신(JVM) 상에서 구동할 때는 `java` 명령어를 수행합니다. 이때 JVM에게 **"어디서 해당 클래스 파일을 찾아야 하는지"** 알려주어야 하며, 이 검색 경로를 지정하는 옵션이 바로 **클래스패스(Classpath, `-cp` 또는 `-classpath`)**입니다.

```bash
# -cp 뒤에 클래스 루트 디렉토리를 적고, 그 뒤에 실행할 메인 클래스의 풀 네임을 입력합니다.
java -cp bin/main com.eomcs.lang.ex03.Exam0100
```

### 1) `-cp bin/main` 의 의미
JVM에게 클래스 수집을 시작할 루트 폴더(Base Directory)를 `bin/main`으로 지정합니다.

### 2) `com.eomcs.lang.ex03.Exam0100` 의 의미
루트 폴더(`bin/main`) 하위에서 패키지 구분을 슬래시(`/`)로 전환하여 `com/eomcs/lang/ex03/Exam0100.class` 파일을 찾아 실행합니다.
* **주의**: 패키지 루트 디렉토리 내부(`bin/main/com/...`)로 직접 진입하여 `java Exam0100`을 실행하면 `NoClassDefFoundError` 에러를 맞닥뜨리게 됩니다. JVM은 패키지 풀 네임 구조를 하나의 클래스 정체성으로 간주하기 때문에 항상 패키지 루트 바깥에서 경로를 지시해야 합니다.

---

## 3. 외부 라이브러리(jar)를 포함하여 컴파일/실행하기

실무에서는 자바 내장 API 외에 외부 라이브러리(jar 파일)를 참조하여 개발하는 경우가 대부분입니다. 터미널 환경에서는 클래스패스에 세미콜론(Windows)이나 콜론(Mac/Linux)을 구분자로 사용하여 여러 디렉토리나 외부 라이브러리 경로를 나열해 줄 수 있습니다.

```bash
# Mac/Linux 환경에서 외부 라이브러리 lib/gson.jar를 클래스패스에 추가하여 실행
java -cp bin/main:lib/gson.jar com.eomcs.lang.ex03.Exam0100
```

CLI 명령어와 클래스패스의 매핑 관계를 명확하게 정립하면 빌드 툴 없이도 어떤 환경에서든 자바 프로그램을 다룰 수 있는 자신감을 가질 수 있습니다.
