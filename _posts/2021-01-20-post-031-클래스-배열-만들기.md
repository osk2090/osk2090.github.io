자바에서 기본 자료형(Primitive Type) 배열뿐만 아니라, 직접 정의한 클래스(참조 타입, Reference Type)를 배열로 다루어야 하는 경우가 매우 흔하게 발생합니다. 이번 포스팅에서는 자바에서 클래스 배열을 올바르게 선언하고 인스턴스를 저장하는 방법과, 이 과정에서 내부적으로 일어나는 메모리 구조(스택과 힙 영역)에 대해 자세히 알아보겠습니다.

## 1. 클래스 배열이란?

클래스 배열(객체 배열)은 동일한 클래스의 인스턴스(객체) 주소들을 가리키는 **참조 변수들의 배열**입니다. 기본 자료형 배열(예: `int[]`, `double[]`)이 배열의 각 칸에 실제 값을 직접 저장하는 것과 달리, 클래스 배열은 객체가 생성된 메모리 주소(참조값)를 저장합니다.

---

## 2. 클래스 배열의 선언 및 생성 2단계

자바에서 클래스 배열을 안전하게 사용하기 위해서는 반드시 **2단계의 초기화 과정**을 거쳐야 합니다. 많은 입문자들이 1단계만 실행하고 객체를 사용하려다 `NullPointerException`을 직면하곤 합니다.

### 예제 클래스 정의
```java
public class Student {
    String name;
    int age;

    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public void showInfo() {
        System.out.println("이름: " + name + ", 나이: " + age);
    }
}
```

### [1단계] 배열 공간 생성 (참조 변수 배열 선언)
배열의 크기를 지정하여 객체들을 저장할 '공간(주소를 담을 그릇)'을 메모리에 먼저 할당합니다.
```java
// Student 객체의 주소 3개를 담을 수 있는 배열 공간을 생성합니다.
Student[] students = new Student[3];
```
* **주의**: 이 시점에서 생성된 것은 `Student` 객체 자체가 아니라, 객체의 주소를 담을 수 있는 변수 3개가 일렬로 연결된 배열입니다. 배열의 각 칸에는 초기값으로 `null`이 채워져 있습니다.

### [2단계] 개별 객체(인스턴스) 생성 및 배열 할당
각 공간에 `new` 키워드를 사용하여 실제 객체를 생성하고 대입해 주어야 합니다.
```java
// 실제 객체를 생성하여 배열의 각 인덱스에 할당합니다.
students[0] = new Student("김철수", 20);
students[1] = new Student("이영희", 22);
students[2] = new Student("박민수", 21);
```

이제 배열의 각 요소를 통해 메소드를 정상적으로 호출할 수 있습니다.
```java
for (int i = 0; i < students.length; i++) {
    students[i].showInfo();
}
```

## 3. 자바 클래스 배열의 메모리 구조 분석

클래스 배열이 메모리 내의 **스택(Stack) 영역**과 **힙(Heap) 영역**에서 어떻게 작용하는지 도식과 함께 이해하면 더 직관적으로 다가옵니다.

1. **`Student[] students` (스택 영역)**
   * 참조 변수인 `students`는 호출 스택 프레임에 로컬 변수로 할당되며, 힙 영역에 생성될 배열 인스턴스의 주소값을 가집니다.
2. **`new Student[3]` (힙 영역 - 배열 인스턴스)**
   * 힙 영역에 3개의 참조값을 보관할 수 있는 크기의 연속된 메모리 공간이 마련됩니다.
   * `students[0]`, `students[1]`, `students[2]` 방이 생기고, 이 방에는 초기값 `null`이 할당되어 있습니다.
3. **`new Student(...)` (힙 영역 - 개별 객체 인스턴스)**
   * `new Student("김철수", 20)` 명령이 실행될 때마다 힙 영역 내 다른 위치에 Student 객체(속성 `name`, `age`를 포함)가 독립적으로 생성됩니다.
   * 그리고 그 생성된 Student 객체의 주소값이 `students[0]` 방에 전달되어 힙 영역 내부에서 다시 힙 영역의 다른 객체를 가리키는(Point) 형태로 구조가 완성됩니다.

이처럼 자바의 객체 배열은 **이중 참조 구조**를 취하기 때문에, 배열의 크기를 만드는 것과 실제 저장할 개별 객체를 인스턴스화하는 것은 별개의 작업임을 인지하는 것이 핵심입니다.

---

## 4. 자주 발생하는 실수: NullPointerException

아래 코드는 클래스 배열을 활용할 때 가장 많이 범하는 오류 사례입니다.

```java
Student[] myStudents = new Student[2]; // 1단계 완료
myStudents[0].name = "에러메이커"; // 에러 발생! NullPointerException
```
* **원인**: `new Student[2]`로 껍데기(배열)만 만들었을 뿐, `myStudents[0]` 공간은 여전히 비어있는 `null` 상태입니다. `null`에 도트(`.`) 연산자를 붙여 속성에 접근하려 했기 때문에 런타임 에러가 터집니다.
* **해결**: 항상 `myStudents[0] = new Student("정상객체", 20);`과 같이 개별 객체 초기화를 선행해야 합니다.

클래스 배열은 객체지향 프로그래밍에서 데이터를 그룹화하여 효과적으로 제어할 수 있는 기본적이면서도 강력한 도구입니다. 메모리 내 주소 흐름을 머릿속에 그리며 코딩을 진행하시면 한결 버그 없는 프로그램을 설계하실 수 있을 것입니다.
