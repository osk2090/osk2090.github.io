소프트웨어를 개발할 때 최초 구현만큼이나 중요한 것이 바로 코드의 유지보수성과 가독성을 높이는 **리팩토링(Refactoring)**입니다. 마틴 파울러의 저서 《리팩터링》에는 코드를 점진적으로 개선하기 위한 수많은 실무 기법들이 소개되어 있습니다. 이번 포스팅에서는 그중에서도 가장 빈번하고 효과적으로 쓰이는 기본 리팩토링 기법 두 가지인 **메서드 추출하기(Extract Method)**와 **임시 변수를 질의 메서드로 바꾸기(Replace Temp with Query)**에 대해 구체적인 자바 예제 코드와 함께 알아보겠습니다.

---

## 1. 메서드 추출하기 (Extract Method)

### 개념
그룹으로 묶을 수 있는 코드 조각이 존재할 때, 해당 코드를 뽑아내어 **의도가 명확히 드러나는 이름을 가진 별도의 메서드**로 만드는 기법입니다.

### Before 리팩토링
아래 코드는 주문 내역을 출력하는 메서드입니다. 영수증 헤더 출력, 상세 내역 계산, 푸터 출력이 하나의 거대한 메서드 안에 섞여 있어 읽기가 까다롭습니다.
```java
public void printOwing(Invoice invoice) {
    // 1. 헤더 출력
    System.out.println("*******************");
    System.out.println("**** Customer Owes ****");
    System.out.println("*******************");

    // 2. 외상값 계산
    double outstanding = 0.0;
    for (Order order : invoice.getOrders()) {
        outstanding += order.getAmount();
    }

    // 3. 세부 정보 출력
    System.out.println("고객명: " + invoice.getCustomer());
    System.out.println("외상값: " + outstanding);
}
```

### After 리팩토링
각 관심사를 독립된 메서드로 추출하여 가독성을 극대화합니다.
```java
public void printOwing(Invoice invoice) {
    printBanner();
    double outstanding = getOutstanding(invoice);
    printDetails(invoice, outstanding);
}

private void printBanner() {
    System.out.println("*******************");
    System.out.println("**** Customer Owes ****");
    System.out.println("*******************");
}

private double getOutstanding(Invoice invoice) {
    double result = 0.0;
    for (Order order : invoice.getOrders()) {
        result += order.getAmount();
    }
    return result;
}

private void printDetails(Invoice invoice, double outstanding) {
    System.out.println("고객명: " + invoice.getCustomer());
    System.out.println("외상값: " + outstanding);
}
```
* **효과**: `printOwing` 메서드의 전체 구조가 단 세 줄의 메서드 호출로 표현되어 코드의 추상화 레벨이 일치하고 가독성이 획기적으로 상승했습니다.

## 2. 임시 변수를 질의 메서드로 바꾸기 (Replace Temp with Query)

### 개념
메서드 안에서 연산된 표현식의 결과를 임시 변수(`temp`)에 임시로 대입해 사용하는 코드가 있다면, 그 표현식 자체를 **독립된 메서드(Query)**로 정의하고 임시 변수에 대한 참조를 메서드 호출로 대체하는 기법입니다.

### Before 리팩토링
```java
public double getPrice() {
    // 임시 변수 basePrice와 discountFactor
    double basePrice = quantity * itemPrice; 
    double discountFactor;
    
    if (basePrice > 1000) {
        discountFactor = 0.95;
    } else {
        discountFactor = 0.98;
    }
    
    return basePrice * discountFactor;
}
```
이 방식은 로직이 복잡해질수록 메서드의 길이가 길어지고, 임시 변수의 생명주기(Scope) 관리가 꼬이기 쉬워집니다.

### After 리팩토링
임시 변수의 계산식을 개별 질의 메서드로 전환합니다.
```java
public double getPrice() {
    return basePrice() * discountFactor();
}

private double basePrice() {
    return quantity * itemPrice;
}

private double discountFactor() {
    return (basePrice() > 1000) ? 0.95 : 0.98;
}
```
* **효과**: 임시 변수가 완전히 제거되어 `getPrice` 메서드가 한 눈에 이해되도록 압축되었습니다. 또한 추출된 `basePrice()`와 `discountFactor()` 메서드는 클래스 내 다른 메서드에서도 재사용할 수 있게 되었습니다.

---

## Summary

* **Extract Method**: 함께 묶일 만한 로직을 추출해 직관적인 이름의 private 메서드로 분리하는 기법.
* **Replace Temp with Query**: 로컬 변수에 특정 수식의 결괏값을 담아두기보다, 해당 수식 자체를 질의 메서드로 추출하여 변수 선언을 제거하는 기법.

이러한 리팩토링 기법들은 객체지향 원칙 중 **단일 책임 원칙(SRP)**을 지키는 견고하고 깔끔한 클린 코드를 만드는 훌륭한 밑거름이 됩니다.
